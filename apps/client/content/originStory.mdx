# Origin Story of Bike4Mind{'\u2122'}

I started working deeply with ChatGPT 3, then 3.5 and then GPT4 and kept discovering deeper potential with LLMs and generative AI when applied to game design.
To deal with the context buffer limitations I iterated on my own and discovered chain of thought, tree of thought, and more...
I even developed variables and function calling in the context window to achieve archaic agent-like behaviors.
I found myself over and over again having to 'spell-cast' by gathering the same context over and over to describe Million on Mars, and the key design paradigms and structure of our game.

The shape of Bike4Mind{'\u2122'} started to form: I wanted a tool where I could upload files: CSVs, JSONs, TXT, even PDFs, or links and have all of these bits of knowledge at my finger tips.
Then I wanted to be able to tap on these bits of Knowledge, one by one and toggle them on or off to be in the context of my prompt.
Some of these are to be background material, others are explicit instructions on how to respond, some on format, some on theme or style.
I wanted to expose all of the LLM parameters like temperature, frequency and presence penalties, and I wanted to expose multiple responses.
I wanted to toggle between models of 3.5 and 4 from OpenAI, and I wanted the freedom to connect to any other model

The 'chat wrappers' did not have all of these features, and while the OpenAI GPT is the best chat client from the major LLM providers, it also did not have all of these features.
I felt the call to try build this tool myself. After two months of work Bike4Mind{'\u2122'} started to work for me, and I was spending more and more time in Bike4Mind{'\u2122'} than in GPT4 - it was time to bring on the team!

## Use Cases for Million on Mars

- Load the 45MB JSON file that holds all of the Game Design Data for Millon on Mars
- Use our own JSON splitter to turn into chunks for each of the design areas: Items, Recipes, Loot Tables, Missions, Achievements, etc
- Vectorize all of these split bits of JSON
- Create explicit instruction prompt templates using the [system] keyword embedded in the prompt instructions to be able to generate the CSV formats for Items, Recipes, etc
- Test and tweak generation of new game content with various LLM parameters, and then be able to save these settings and [Knowledge](knowledge) to create [Tools](tools)
- Export these creations to the Million on Mars Game Design Editor (GDE) and paste them in, review and create content branches with much higher quality, velocity and lower labor costs
- Take these game design updates and generate Discord, Twitter, player guide and in-game updates
- All to deliver more content and gameplay, faster, higher quality and reduce the labor required, and have human bandwidth focused on true, design and creative work instead of the rote detail work
