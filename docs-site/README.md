# Bike4Mind Documentation Site

The public documentation site for Bike4Mind, built with [Docusaurus](https://docusaurus.io/) and deployed with [SST](https://sst.dev/) to docs.bike4mind.com.

## Local development

From this directory (standalone workspace - use `--ignore-workspace`):

```bash
pnpm install --frozen-lockfile --ignore-workspace
pnpm start        # dev server on http://localhost:3010
pnpm build        # production build into build/
pnpm serve        # serve the production build
```

Or from the repo root: `pnpm docs:dev` / `pnpm docs:build`.

## Adding content

1. Create a Markdown file in the right `docs/` subdirectory.
2. Add frontmatter (`title`, `description`, `sidebar_position`).
3. The sidebar is autogenerated from the filesystem; ordering and labels come from each directory's `_category_.json` and per-file `sidebar_position`.

The build fails on broken links (`onBrokenLinks: throw`), so run `pnpm build` before pushing.

## Content rules

This site is public. No customer or partner names, no internal issue references, no employee names, no real infrastructure identifiers (account IDs, ARNs, bucket or cluster names, hosted-zone IDs) - use obvious placeholders. Self-host commands must match SELF_HOST.md at the repo root exactly.

## Deployment

Deploys are handled by `.github/workflows/deploy-docs.yml`: pushes to `main` touching `docs-site/` deploy staging (docs.staging.bike4mind.com); production (docs.bike4mind.com) is a manual workflow dispatch behind the Production environment. A sensitive-term scan gates every deploy.
